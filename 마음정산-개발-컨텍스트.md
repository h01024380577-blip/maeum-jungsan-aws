# 마음정산 (HeartBook) 개발 컨텍스트

## 프로젝트 개요
- 한국 경조사(결혼식, 부고, 생일) 축의금/조의금 관리 앱
- AI로 청첩장/부고장 링크를 분석하여 자동 입력
- 배포: https://maeum-jungsan.vercel.app
- GitHub: https://github.com/h01024380577-blip/maeum-jungsan

## 기술 스택
- Next.js 16 (App Router, Turbopack)
- React 19, TypeScript, Tailwind CSS v4
- Zustand (클라이언트 상태) + Supabase 직접 연동 (entries/contacts 테이블)
- Prisma 6 ORM + Supabase PostgreSQL (NextAuth용 User/Account/Session + Event/Transaction)
- NextAuth.js 4 (카카오 로그인)
- Gemini 2.5 Flash (AI 분석), Vitest (테스트 32개)
- Jina Reader (SPA 렌더링 텍스트 추출)

## 데이터 레이어 (이중 구조)
- Zustand + Supabase 직접: entries, contacts 테이블 (기존 CRUD, DEVICE_ID 기반)
- Prisma + NextAuth: User, Account, Session, VerificationToken, Event, Transaction (카카오 로그인 + 향후 DB 연동)
- 두 레이어가 같은 Supabase PostgreSQL DB를 공유하지만 별도 테이블 사용

## 완료된 작업

### 1. URL 분석 고도화 (서버사이드 파싱)
- 기존: 클라이언트에서 Gemini에 URL 문자열만 전달 → hallucination 발생
- 개선: 3단계 파이프라인
  - Phase 1: 서버에서 HTML fetch → cheerio로 og 메타태그 + 본문 추출 → Gemini 분석
  - Phase 2: Jina Reader (r.jina.ai)로 JS 렌더링된 텍스트 추출 → Gemini 분석 (SPA 대응)
  - Phase 3: Gemini urlContext 도구 (최후 수단)
- 파일: app/api/parse-url/route.ts, src/lib/parseUrl.ts, src/lib/fetchPage.ts
- 테스트: src/lib/parseUrl.test.ts (13개), src/lib/fetchPage.test.ts (7개)
- 검증: theirmood.com 링크로 이름/날짜/장소/계좌 전부 정확 추출 확인
- Gemini API 한도 초과 시 "무료 분석 한도를 모두 이용하셨습니다" 토스트 표시

### 2. 카카오 소셜 로그인
- NextAuth.js + KakaoProvider + PrismaAdapter
- 파일: src/lib/auth.ts, app/api/auth/[...nextauth]/route.ts
- 카카오 앱 ID: 1417009
- 플랫폼 키(REST API): [Vercel 환경변수 참조]
- Redirect URI: https://maeum-jungsan.vercel.app/api/auth/callback/kakao
- 동의항목: 닉네임(필수), 프로필사진(선택)
- 로그인 UI: HomeTab 헤더에 카카오 노란색 버튼 + 프로필 표시

### 3. 데이터베이스 스키마
- Prisma 6 (v7은 Vercel 서버리스 .prisma/client/default 모듈 누락으로 다운그레이드)
- prisma/schema.prisma:
  - User (emailVerified 포함 — NextAuth 필수)
  - Account (refresh_token_expires_in 포함 — 카카오 전용)
  - Session, VerificationToken
  - Event (eventType, uiTheme: FUNERAL→SOLEMN 자동 설정, confidence)
  - Transaction (금액/내역, isPaid)
- Supabase 직접 테이블 (Zustand용):
  - entries (event_type, target_name, amount 등 — snake_case)
  - contacts (name, phone, relation 등)
  - prisma db push로 삭제된 후 SQL Editor에서 재생성함
- DB 호스트: aws-1-ap-northeast-1.pooler.supabase.com:5432

### 4. 경조사 API (Prisma 기반, 향후 연동용)
- POST /api/events — 인증된 사용자의 경조사 + 거래내역 저장
- GET /api/events — 내 경조사 목록 조회
- 파일: app/api/events/route.ts, src/lib/events.ts
- 테스트: src/lib/events.test.ts (9개), src/lib/auth.test.ts (3개)

### 5. UI 수정
- 달력 금액 표시에서 '원' 제거
- 홈화면 요약 카드: 받은→받음, 보낸→보냄
